## 核心原則

| 原則 | 說明 |
|------|------|
| **一步一動作** | 每個步驟只描述一個操作，不要合併多個動作 |
| **用關鍵字開頭** | 步驟必須以系統可辨識的「動作關鍵字」開頭（見下表） |
| **指定可見文字** | 目標元素請使用**頁面上實際看得到的文字**，而非功能描述 |
| **避免模糊描述** | ❌「操作選單」→ ✅「點選『全宗瀏覽』」 |
| **網址要完整** | 導航步驟請寫出完整路徑或 URL |

## 文件結構範本

Word 文件建議使用以下**表格格式**，系統會自動辨識：

| 項目序 | 測試項目 | 操作步驟 | 預期結果 | 測試結果 |
|--------|----------|----------|----------|----------|
| 1 | 首頁載入 | 前往 https://example.com | 頁面正常顯示 | □通過 / □失敗 |

### 文件開頭請加入 Meta 資訊

在文件最前面加入以下資訊，系統會自動擷取：

```
測試名稱：XXX系統前台功能測試
網址：https://your-website.com
環境：正式環境 / 測試環境
撰寫者：王大明
```

## 動作關鍵字對照表

系統會根據步驟文字開頭的「關鍵字」判斷要執行什麼操作。
**請務必使用以下關鍵字開頭撰寫每個步驟。**

### 導航（前往頁面）

| 關鍵字 | 範例 |
|--------|------|
| `前往` | 前往 https://example.com/about |
| `開啟` | 開啟 /search 頁面 |
| `進入` | 進入 https://example.com/login |
| `瀏覽` | 瀏覽 /products 頁面 |
| `訪問` | 訪問 https://example.com |

> ⚠️ **重要**：目標請寫 **URL 或路徑**，而非頁面名稱。
>
> - ✅ `前往 https://example.com`
> - ✅ `前往 /search`
> - ❌ `前往首頁`（系統無法判斷「首頁」是哪個 URL）

### 點擊

| 關鍵字 | 範例 |
|--------|------|
| `點選` | 點選「全宗瀏覽」 |
| `點擊` | 點擊「登入」按鈕 |
| `按下` | 按下「送出」 |
| `切換` | 切換「全宗範圍」 |
| `展開` | 展開「進階搜尋」 |

> ⚠️ **重要**：目標請使用**頁面上實際可見的文字**。
>
> - ✅ `點選「全宗瀏覽」`（頁面上有「全宗瀏覽」這幾個字）
> - ❌ `點選主選單`（如果頁面上沒有「主選單」三個字）
>
> **如何找到正確的文字？** 打開瀏覽器，看你要點的元素上面寫什麼字，就用那個字。

### 輸入文字

| 關鍵字 | 範例 |
|--------|------|
| `輸入` | 輸入「南方資料館」關鍵字 |
| `填入` | 填入「test@example.com」到 email 欄位 |
| `填寫` | 填寫「王大明」到姓名欄位 |
| `鍵入` | 鍵入「12345」到密碼欄位 |

> 📝 **兩種寫法：**
>
> **寫法一：指定欄位**（更精確）
> ```
> 輸入「test@example.com」到 #email
> 填入「密碼123」到 input[name=password]
> ```
>
> **寫法二：搜尋框快捷寫法**
> ```
> 輸入「南方資料館」關鍵字
> ```
> 系統會自動尋找搜尋框（input[type=search], #search 等）

### 選擇下拉選單

| 關鍵字 | 範例 |
|--------|------|
| `選擇` | 選擇「台北市」於 #city 下拉選單 |
| `選取` | 選取「2026」從年份選單 |

### 驗證（檢查頁面是否正確）

| 關鍵字 | 範例 |
|--------|------|
| `確認` | 確認「歡迎回來」可見 |
| `驗證` | 驗證 網址 包含 /dashboard |
| `檢查` | 檢查 文字 包含「登入成功」 |

### 其他動作

| 關鍵字 | 範例 |
|--------|------|
| `等待` | 等待 3000（毫秒） |
| `截圖` | 截圖 login_page |
| `滑鼠移至` | 滑鼠移至「使用者圖示」 |
| `捲動` | 捲動 500（像素） |

## 步驟撰寫範例

### ✅ 好的寫法（系統可正確執行）

| 項目序 | 測試項目 | 操作步驟 | 預期結果 |
|--------|----------|----------|----------|
| 1 | 首頁載入 | 前往 https://tais.ith.sinica.edu.tw/ | 頁面正常顯示 |
| 2 | 全宗瀏覽 | 點選「全宗瀏覽」 | 顯示全宗列表 |
| 3 | 關鍵字搜尋 | 輸入「南方資料館」關鍵字；點選「搜尋」 | 出現搜尋結果 |
| 4 | 切換分類 | 切換「檔案」 | 頁面呈現檔案分類 |

> 💡 多個步驟可用 **分號（；）** 或 **箭頭（→）** 分隔，系統會自動拆分。

### ❌ 不好的寫法（系統無法正確執行）

| 操作步驟 | 問題 | 修正建議 |
|----------|------|----------|
| 進入首頁 | 「首頁」不是 URL | → `前往 https://example.com/` |
| 點選主選單→全宗瀏覽 | 「主選單」可能不是頁面上的文字 | → `點選「全宗瀏覽」` |
| 點選任一筆單件內容頁 | 「任一筆」太模糊，系統不知道選哪個 | → `點選第一個搜尋結果的標題連結` |
| 使用不同裝置開啟網頁 | 這是手動測試描述，非自動化步驟 | → `前往 https://example.com/`（系統會以桌面瀏覽器測試） |
| 單一IP於一定時間內大量檢索 | 這是測試情境描述，非步驟 | → 拆成多個步驟：重複輸入關鍵字並搜尋 |

## 元素指定方式

當需要精確指定頁面元素時，可使用以下方式：

### 方式一：使用可見文字（推薦，最簡單）

```
點選「登入」
點選「全宗瀏覽」
確認「歡迎回來」可見
```

### 方式二：使用 CSS 選擇器（更精確）

適用於有多個相同文字的按鈕，或元素沒有文字時：

```
點選 #login-button
點選 .nav-menu > li:first-child
輸入「test」到 input[name=search]
```

### 方式三：使用 HTML ID

```
點選 #submit-btn
輸入「test」到 #search-input
```

> 💡 不確定元素名稱？請在瀏覽器按 F12 開啟開發者工具，滑鼠移到元素上查看。
> 或請工程師協助提供元素 ID / CSS 選擇器。

## 完整範例文件

以下是一份完整的測試文件範例，可直接複製到 Word 中使用：

---

```
測試名稱：TAIS 臺灣史檔案資源系統前台測試
網址：https://tais.ith.sinica.edu.tw/
環境：正式環境
撰寫者：王大明
```

### 個案一：前台介面

| 項目序 | 測試項目 | 操作步驟 | 預期結果 | 測試結果 |
|--------|----------|----------|----------|----------|
| 1 | 首頁載入 | 前往 https://tais.ith.sinica.edu.tw/ | 頁面正常顯示，有導覽列 | □通過 / □失敗 |
| 2 | 全宗瀏覽 | 點選「全宗瀏覽」 | 顯示全宗列表頁面 | □通過 / □失敗 |
| 3 | 影像瀏覽 | 點選 第一筆搜尋結果的連結；確認「影像瀏覽」可見 | 開啟影像瀏覽器 | □通過 / □失敗 |
| 4 | 響應式版面 | 前往 https://tais.ith.sinica.edu.tw/；截圖 homepage | 擷取首頁畫面 | □通過 / □失敗 |

### 個案二：搜尋功能

| 項目序 | 測試項目 | 操作步驟 | 預期結果 | 測試結果 |
|--------|----------|----------|----------|----------|
| 1 | 關鍵字搜尋 | 前往 https://tais.ith.sinica.edu.tw/；輸入「南方資料館」關鍵字；點選「搜尋」 | 出現包含「南方資料館」的搜尋結果 | □通過 / □失敗 |
| 2 | 空白搜尋 | 前往 https://tais.ith.sinica.edu.tw/；點選「搜尋」 | 顯示提示訊息或全部結果 | □通過 / □失敗 |

## 常見 Q&A

**Q：分號和箭頭可以混用嗎？**
A：可以。 `前往 /search；輸入「test」關鍵字→點選「搜尋」` 都會被正確拆分。

**Q：我不知道元素的 CSS 選擇器怎麼辦？**
A：用頁面上可見的文字即可。例如按鈕上寫「搜尋」，就寫 `點選「搜尋」`。

**Q：手動測試的步驟（如切換裝置）怎麼寫？**
A：自動化測試用桌面瀏覽器執行，無法真的切換裝置。建議改為 `前往 URL` + `截圖`，
用截圖方式記錄頁面狀態即可。

**Q：怎麼測試需要登入的頁面？**
A：在前面步驟加入登入流程：
```
1. 前往 https://example.com/login
2. 輸入「user@example.com」到 #email
3. 輸入「password123」到 #password
4. 點選「登入」
5. 確認 網址 包含 /dashboard
```

**Q：一個步驟很複雜，一行寫不下怎麼辦？**
A：拆成多行，每行一個動作。**不要**把多個動作塞在一格裡。

## 關鍵字速查卡

| 你想做什麼 | 寫法 |
|------------|------|
| 開啟網頁 | `前往 URL` |
| 按按鈕 | `點選「按鈕文字」` |
| 按連結 | `點選「連結文字」` |
| 打字到輸入框 | `輸入「文字」到 #元素ID` |
| 搜尋關鍵字 | `輸入「關鍵字」關鍵字` |
| 選下拉選單 | `選擇「選項」於 #元素ID` |
| 確認文字存在 | `確認「文字」可見` |
| 確認網址對 | `驗證 網址 包含 /path` |
| 等一下 | `等待 3000` |
| 拍照存證 | `截圖 名稱` |
| 滑動頁面 | `捲動 500` |
