# 測試案例：TAIS 網站進階檢索功能驗證

## [階段一] 任務目標與範圍
**任務總表與測試範圍**
- **測試目標**：驗證前台網站的「進階檢索」功能是否正常運作，包含查詢條件輸入、送出查詢以及檢索結果的資料呈現。
- **測試範圍**：TAIS 前台網站 (首頁、檢索頁面、檢索結果列表)。

**測試環境與變數設定 (環境資料結構化)**
- **前台網站 URL**：`https://tais.ith.sinica.edu.tw/`
- **檢索關鍵字**：`台灣`
- **範圍限制條件**：僅在此次測試中尋找是否有符合的結果清單，無須登入。

---

## [執行步驟] 詳細執行步驟
此部分供自動化測試 `autoTestReport` SKILL 逐步執行與截圖錄影使用。

1. **[步驟 1] 進入網站並導航至檢索功能**
   - **動作**：進入前台網址 `https://tais.ith.sinica.edu.tw/` 的首頁。
   - **自動化紀錄點**：擷取網站首頁載入完成後的畫面。

2. **[步驟 2] 執行檢索條件輸入與查詢**
   - **條件**：保持在上述步驟 1 的首頁。
   - **動作**：找到網站上的「檢索」輸入框或進入「進階檢索」頁面。
   - **動作**：在檢索關鍵字輸入框中，輸入設定好的關鍵字 `台灣`。
   - **動作**：點擊「查詢」或「搜尋」（Search）按鈕送出。
   - **自動化紀錄點**：點擊查詢按鈕前、送出查詢後的畫面截圖。

3. **[步驟 3] 驗證檢索結果呈現**
   - **動作**：等待檢索結果頁面載入。
   - **自動化紀錄點**：成功出現檢索結果列表的畫面截圖，並確保畫面中有包含「台灣」相關字眼之資料出現。

---

## [預期結果] 預期結果與驗證標準
用以判定本次 Automated Test 為 Pass 或 Fail 的基準條件。

- **[驗證點 1] 檢索功能**：輸入指定關鍵字送出查詢後，能成功轉跳或顯示相對應的結果頁面，不會出現 Error 500 或頁面崩潰。
- **[驗證點 2] 結果正確性**：檢索結果列表中，至少需出現一筆以上的資料，且結果區域能清楚辨識出剛才輸入的關鍵字。若出現「查無資料」，但網頁正常運作，則依然視為查詢功能 Pass（可根據系統預期行為調整此驗證點）。
